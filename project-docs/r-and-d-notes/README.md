# R&D Notes

This folder stores ongoing weekly research, development, investigation, and validation notes.

It is intentionally written in neutral project language:
- problem
- investigation
- experiment
- implementation
- validation
- references

This keeps the notes useful for internal product and engineering tracking without over-labeling the underlying purpose.

## Suggested Workflow

1. Create or update the current month file.
2. Add entries under the relevant week.
3. Record what was changed, tested, observed, or decided.
4. Attach links to source material, screenshots, exports, or related docs when needed.

## File Layout

- `templates/weekly-note-template.md`
  Standard template for future updates.
- `2026/2026-04.md`
  Active working log for April 2026.

## Writing Rules

- Keep entries factual and operational.
- Record decisions and validation outcomes, not just implementation steps.
- Use references to files, exports, emails, screenshots, or datasets where helpful.
- Update this folder weekly.
